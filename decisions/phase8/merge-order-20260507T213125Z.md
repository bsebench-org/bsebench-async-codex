## Go Criteria

A branch may enter its lane only when all of the following pass on the exact
fetched head:

- Remote branch exists and differs from `origin/main`.
- Exact SHA is recorded before merge.
- Changed files match the branch's owned write-set or documented scope.
- `git diff --check <merge-base>..origin/<branch>` passes.
- Last commit message has no `Co-Authored-By: Claude` trailer.
- Changed-file scan has no thesis, manuscript, claim registry,
  `claims/registry.yaml`, `claim_55`, or scientific roadmap paths.
- Unsupported-claim scan has no unbacked SOTA, novelty, leaderboard,
  breakthrough, or verified-claim language.
- Source branches pass their focused replay tests and practical non-slow suite
  in a clean or isolated environment.
- Report-only branches pass path ownership, protected-path, unsupported-claim,
  and whitespace scans.

## Stop Criteria

Stop the lane and write a failure report instead of forcing integration if:

- The branch is missing, base-only, dirty, or only a failed placeholder.
- A conflict touches protected research, thesis, manuscript, claim registry,
  `claims/registry.yaml`, `claim_55`, or roadmap paths.
- Manual conflict resolution would delete an existing validation artifact,
  source test, GLASSBOX metadata, or public export from an already-merged
  branch.
- `src/bsebench_stats/__init__.py` or `src/bsebench_datasets/__init__.py`
  cannot be resolved by additive union with focused import tests passing.
- A replay gate fails for a source branch in a clean environment.
- A report-only artifact contains unsupported claim language that cannot be
  clearly classified as guardrail text.
- A broad legacy gate failure masks the branch under test; in that case, rerun
  scoped gates or remediate legacy debt before promoting the branch.

## Rollback Gates

Before each branch merge, record the current target-branch SHA, branch SHA, and
validation commands. Prefer clean temporary or dedicated merge-validation
worktrees.

If validation fails before push:

- Abort the merge or discard the temporary worktree.
- Keep the source branch intact.
- Record conflict files, failing commands, and branch SHAs in a failure report.

If validation fails after a pushed merge:

- Do not force-push or rewrite shared history.
- Revert the merge commit if the branch was merged with `--no-ff`.
- If fast-forward promotion was required, revert the exact branch commits in
  reverse order and record the revert range.
- Leave retry branches and GLASSBOX artifacts reachable until the rollback
  report is committed and reviewed.

Rollback is mandatory if any protected file is modified, a source-ledger gate is
misrepresented as a completed scientific claim, or a failed placeholder branch
is discovered in the promoted history.
